# TB-31: CLI wrapper: mutation commands (create/edit/mv/close/regenerate)

**Type:** feature
**Priority:** P1
**Size:** S
**Module:** gui
**Tags:** milestone-m3,cli-bridge,mutations
**Branch:** —
**Parent:** TB-3

## Goal

Extend gui/internal/cli with typed wrappers for tb create, edit, mv, close, regenerate so BoardService can exec them safely.

## Acceptance Criteria

- [ ] `gui/internal/cli` exports typed wrappers: `Create`, `Edit`, `Move`, `Close`, `Regenerate`
- [ ] `Create` parses the CLI's text output (`Created <path>`) and returns the new task ID derived from the filename (regex `<PREFIX>-(\d+)\.md`); `Edit`, `Move`, `Close`, `Regenerate` use exit code only
- [ ] Each wrapper returns a typed error distinguishing missing-binary, board-not-found, validation, and unknown-failure cases (parse stderr for known patterns; fall back to `ErrUnknown` with the raw stderr attached)
- [ ] Unit tests cover happy path and at least one structured error path per command (table-driven against a fake `tb` binary using the same harness as `cli_test.go`)
- [ ] No direct file writes — all mutations go through `exec.Command`
- [ ] Follow-up backlog: extending `tb` mutations with `--json` output is tracked separately (not blocking this task); the parse-text contract here is good enough for M3
