# TB-119: Worktree flow: acceptance criteria as integration tests

**Type:** feature
**Priority:** P1
**Size:** L
**Module:** cli
**Branch:** тАФ
**Parent:** TB-109

## Goal

Turn every acceptance criterion from ┬з 10 of the worktree-isolation spec into an automated integration test. Each test sets up a real git repo + board, drives `tb` commands (or simulates daemon code paths), and asserts on disk/git state.

## Context

Spec section: ┬з 10 (all ACs), reused as test cases.

Many ACs can land alongside their feature task (e.g., AC9b's atomicity check in TB-113), but consolidating into one umbrella task ensures the full matrix gets coverage and that integration scenarios spanning multiple subsystems (AC8 series) are tested as units.

## Acceptance Criteria

Each spec AC becomes a Go test:

**Basic flow:**
- [ ] AC1 тАФ fresh board, `tb start` produces all artifacts.
- [ ] AC2 тАФ `tb edit` from worktree (without `TB_BOARD_PATH`) hits canonical board.

**Clean merge:**
- [ ] AC3a тАФ main on `<merge_target>`, common case FF.
- [ ] AC3b тАФ main on unrelated branch, ref-push FF.
- [ ] AC3c тАФ dirty `<board>/` in main does not block merge.
- [ ] AC3d тАФ uncommitted user code conflicts with FF diff тЖТ propagated git error.
- [ ] AC3e тАФ external worktree on `<merge_target>` тЖТ case C refusal.

**Conflict + retry:**
- [ ] AC4 тАФ induced conflict leaves rebase mid-flight, `MergeStatus: conflict`.
- [ ] AC5 тАФ `git rebase --continue` + `tb merge` finalizes cleanly.

**Concurrency + safety:**
- [ ] AC6 тАФ two tasks parallel with `max_workers=2`.
- [ ] AC7 тАФ cancel preserves worktree.
- [ ] AC9 тАФ branch collision errors out clean.
- [ ] AC9b тАФ injected failure mid-creation тЖТ task stays in `backlog/`.
- [ ] AC10 тАФ dirty worktree тЖТ `dirty_worktree`.
- [ ] AC11 тАФ `worktree clean` refuses unmerged commits without `--force`.

**Preflight:**
- [ ] AC12 тАФ submodule detection.
- [ ] AC13 тАФ path conflict detection.
- [ ] AC14 тАФ pre-push hook rejects publishing `task/*`.

**Recovery scenarios:**
- [ ] AC8a тАФ rebasing crash тЖТ `conflict`.
- [ ] AC8b тАФ finalizing post-FF crash тЖТ idempotent finalize, NOT `conflict`.
- [ ] AC8c тАФ cleanup crash тЖТ idempotent cleanup.
- [ ] AC8d тАФ backlog-with-active-worktree тЖТ file move completed.
- [ ] AC8e тАФ ancestry check after subsequent merge.
- [ ] AC8f тАФ target drift during manual conflict resolution тЖТ re-rebase loops cleanly.
- [ ] AC8g тАФ corrupt sidecar тЖТ tb doctor entry, no auto-repair.
- [ ] AC8h тАФ `rebase-apply` backend probed.
- [ ] AC8i тАФ user archive during recovery window тЖТ respected.

**Non-functional:**
- [ ] AC15 тАФ typical `tb done` < 2s on uncontended board.
- [ ] AC16 тАФ four concurrent `tb done` serialize via `.merge.lock`, no deadlock.

**Migration (┬з 8):**
- [ ] Existing in-progress task (no sidecar) + `worktrees.enabled: true` тЖТ `tb done` uses degraded move-only path; no rebase attempted, exit success.
- [ ] `tb worktree adopt <ID>` on a pre-mode in-progress task тЖТ creates worktree and sidecar; next `tb done` runs the full TB-115 flow.
- [ ] Flipping `worktrees.enabled: false тЖТ true` on an existing board with no pre-existing `.tb-worktrees/` runs preflight successfully and writes gitignore patterns.
