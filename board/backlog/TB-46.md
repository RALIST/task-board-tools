## Goal

Stand up `gui/app/agent_service.go` and implement the simplest method first: assign or clear the agent for a task. Pure backend — the Card badge and the drawer dropdown live in TB-50.

## Acceptance Criteria

- [ ] `gui/app/agent_service.go` defines `AgentService` with a constructor that accepts the same `cli.Client` (or mutations wrapper) used by `BoardService`
- [ ] `AssignAgent(ctx, id, agent string) error`:
  - Validates `agent ∈ {"", "none", "claude", "codex"}`; rejects everything else with `ErrAgentNotSupported`
  - Treats `""` and `"none"` as "clear the field"; resolves to the right `tb edit -a` invocation in both cases
  - Delegates the mutation through the mutations wrapper from TB-31 (`exec tb edit <ID> -a <agent>` or `-a none`)
- [ ] Typed errors: `ErrAgentNotSupported`, `ErrTaskNotFound` (mapped from the mutations wrapper's existing error classification)
- [ ] Wails bindings regenerated: `AgentService` and `AssignAgent` visible in `gui/bindings/`
- [ ] Unit tests cover happy path (stub `tb` echoing the args), invalid agent, clear-by-none, task-not-found error mapping
- [ ] **Real-tb integration test** (mirrors the `EditTaskBody` test pattern from TB-33): against a tmpdir board with a real `tb` binary, call `AssignAgent("TB-X", "claude")`; assert `tb show TB-X` stdout contains `**Agent:** claude`. Then `AssignAgent("TB-X", "none")`; assert `tb show TB-X` no longer shows `**Agent:** claude`. This is the on-disk truth that backs F4.1 "reload — assignment persists"
