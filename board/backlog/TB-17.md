# TB-17: Scaffold gui/ Wails3 + SvelteKit project with single-instance lock

**Type:** feature

**Size:** S

**Tags:** milestone-m2,scaffold
**Branch:** —

## Goal

Run wails3 init -t sveltekit-ts in gui/, add ./gui to the root go.work, enable the Wails3 single-instance feature, and verify wails3 dev launches a blank window. Foundation for all other M2 frontend/backend work.

## Context

Bootstraps the gui/ module. After this lands, all subsequent M2 tasks have a real project to land code in. Single-instance lock is included here because it is a Wails3 platform feature wired during init — far cheaper to set up at scaffold time than to retrofit later, and it protects the daemon uniqueness invariant in M5.

## Acceptance Criteria

- [ ] `wails3 init -t sveltekit-ts` produces a clean `gui/` tree
- [ ] Root `go.work` updated with `use ./gui`
- [ ] `go build ./gui` from the repo root succeeds
- [ ] `cd gui && wails3 dev` launches a blank window and exits cleanly on Ctrl+C
- [ ] Wails3 single-instance feature enabled — second `wails3 dev` invocation either focuses the first window or refuses to start
- [ ] `gui/go.mod` pins the Wails3 tag confirmed in TB-16
- [ ] Repo `.gitignore` updated to exclude `gui/frontend/node_modules/`, `gui/build/`, `gui/.wails/` (or equivalents per the scaffold)
- [ ] App binary name and bundle identifier set to `tb-gui` in the Wails config
