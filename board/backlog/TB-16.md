## Goal

Run wails3 doctor on the dev machine, document the working Wails3 tag + Go version, and pin them in gui/go.mod and docs/ARCHITECTURE.md. Wails3 is alpha and the plan flags Go 1.26.1 compatibility as a known risk that must be resolved before any other M2 work.

## Context

Wails3 is in alpha and the plan flags Go 1.26.1 compatibility as risk #1 ("Wails3 alpha + Go 1.26.1 compatibility — обязательно проверить `wails3 doctor` в M2 до полноценной разработки. Если не работает — pin Wails3 на конкретный tag или временно downgrade Go."). This is the first thing to validate so the rest of M2 doesn't get blocked mid-way. Also note that Wails brings a CGo toolchain dependency (gcc/clang) — surface that in the docs.

## Acceptance Criteria

- [ ] `wails3 doctor` runs cleanly on the dev machine and the output is captured in the task log
- [ ] If `doctor` fails on current Go, a working Wails3 git tag is identified and pinned (record both Go version and Wails3 tag)
- [ ] CGo dependencies (gcc/clang) documented as a build prerequisite in docs/ARCHITECTURE.md
- [ ] docs/ARCHITECTURE.md updated: confirmed Wails3 tag, supported Go versions, OS support note (macOS + Linux for MVP; Windows = open question per plan risk #5)
- [ ] No code committed to `gui/` yet — this is purely a spike
