# TB-114: Agent daemon: run agent inside worktree

**Type:** feature
**Priority:** P1
**Size:** S
**Module:** gui
**Branch:** —
**Parent:** TB-109

## Goal

When the daemon picks up a `queued` task in a worktree-enabled board, it must **ensure** the worktree exists (idempotent fast-path from TB-113) **before** spawning the agent, then run the agent with `cwd = <worktrees.path>/<ID>` and `TB_BOARD_PATH = <canonical>`. The daemon must NOT fall back to running the agent in the main checkout when worktree mode is on — that's exactly the isolation breach this whole epic exists to prevent.

## Context

Spec sections: § 4.5 ("Idempotency for daemon use — `EnsureWorktree(ID)`"), § 4.6.

The original draft of this task allowed a fallback to `cwd = <repo root>` when no sidecar was present. Codex flagged that as keeping the isolation hole open in the common case (queued task created before worktree mode was turned on, or sidecar got cleaned and never re-created).

## Acceptance Criteria

- [ ] When `worktrees.enabled: true` and the daemon dequeues a task, it calls `EnsureWorktree(ID)` (TB-113's idempotent path) **before** spawning the agent process:
  - If sidecar `state: "active"` + worktree exists → reuse.
  - If sidecar is missing OR `state: "creating"` → run the full TB-113 create flow (which itself handles preflight + locking + rollback).
  - If the create flow fails → the task is left in its current status (no agent spawn), `AgentStatus: failed` with reason `worktree_create_failed`, error surfaced via the existing failure plumbing.
- [ ] The spawned agent process gets:
  - `cwd = <repo root>/<worktrees.path>/<ID>` (resolved from `.tb.yaml`, never hardcoded).
  - `TB_BOARD_PATH = <absolute path to canonical board>` (resolved from main checkout's `.tb.yaml`).
- [ ] When `worktrees.enabled: false` (existing boards that haven't opted in): no behavior change — `cwd = <repo root>`, no `TB_BOARD_PATH`. The fallback is gated **on the board config**, not on "does a sidecar happen to exist."
- [ ] Agent prompts (existing `implement.md` / `groom.md`) gain a small footer: "You are operating inside a git worktree at `<cwd>`. Commit your changes to the current branch. Do not switch branches. Do not run `git push`. The canonical task board is at `$TB_BOARD_PATH` — use `tb` normally."
- [ ] Existing M5 agent run flow (queue, JSONL events, log files) keeps working.
- [ ] Smoke test: with `worktrees.enabled: true`, queue a fresh task that has **no sidecar**, run a trivial agent — verify the daemon creates the worktree first, then the agent runs in it.
- [ ] Smoke test: with `worktrees.enabled: false`, queue and run a task — verify `cwd` is repo root and no worktree gets created.
