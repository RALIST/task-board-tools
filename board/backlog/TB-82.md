## Goal

Add a native macOS application menu via Wails3's menu API:
- **File** — Open board… (Cmd+O), Open Recent › (dynamic submenu populated from `ListRecentBoards()`), Settings… (Cmd+,) → opens TB-81 panel, Quit (Cmd+Q)
- **View** — Reload board (Cmd+R), Show Archived toggle
- **Help** — About tb-gui, Open docs / project README

## Context

Per TB-7's groomed Goal, this is a **native application menu** — distinct from the existing empty-state recent-board list inside the window (TB-2). Recent boards are already persisted at `$XDG_CONFIG_HOME/tb-gui/recent.json` and surfaced by `SettingsService.ListRecentBoards()`; this task wraps them in a native submenu.

**Spec-anchor note**: `docs/FEATURES.md` § M7 currently lists F7.1 (settings UI), F7.2 (shortcuts), F7.3 (tray) — but no application-menu feature. TB-7's parent body adds the menu to M7's polish scope. TB-85 (docs flip) must introduce **F7.4 — Native application menu** with the AC list below, so the docs catch up before this task ships. Wails3 alpha menu API confirmed present at `pkg/application/menu.go`, `pkg/application/menu_manager.go`.

**Probe-first risk**: Wails3 alpha menu API surface is unstable (R1 in `IMPLEMENTATION.md`). Start with a 1-item File menu that toggles a console.log, confirm it renders on macOS, *then* expand. If the alpha API is missing primitives (e.g. dynamic submenu rebuild on `recent.json` change), document the gap in the task log and ship the static menu for M7 — defer dynamic refresh to a follow-up.

The Open Recent submenu rebuilds whenever `recent.json` changes. Easiest hook: a Wails event the SettingsService emits after `rememberBoard` (e.g. `recents:changed`); the menu listener calls `ListRecentBoards()` and rebuilds the submenu.

## Acceptance Criteria

- [ ] Native menu visible on macOS with File / View / Help top-level items (Linux: same menu in window-attached mode if Wails3 alpha supports it; otherwise document and defer)
- [ ] **File → Open board…** triggers `PickBoardDialog` → `OpenBoard` (same path as the empty-state picker)
- [ ] **File → Open Recent ›** lists the last N (≤10) entries from `ListRecentBoards()`; clicking one calls `OpenBoard(projectRoot)`
- [ ] Submenu rebuilds when a new board is opened (a `recents:changed` Wails event after `rememberBoard`, listened to by the menu controller)
- [ ] **File → Settings…** opens TB-81's settings panel
- [ ] **View → Reload board** triggers `BoardService.Regenerate` then a `board:reloaded` emit
- [ ] **Help → About** shows a simple dialog with version + project URL
- [ ] Standard accelerators wired: Cmd+O, Cmd+,, Cmd+Q, Cmd+R
- [ ] Probe step documented in the PR description: 1-item menu confirmed working before expansion
