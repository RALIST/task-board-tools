## Goal

Add gui/internal/cli/cli.go wrapping exec.CommandContext('tb', ...). Discovers the tb binary in PATH (and configurable absolute path), handles non-zero exit codes, returns parsed JSON for read commands, and routes stderr to logs. Foundation for BoardService, SettingsService, and AgentService.

## Context

Every Wails service that talks to the CLI goes through this wrapper, so the cross-cutting concerns (binary discovery, exit-code handling, JSON parsing, stderr routing, ctx cancellation) live in one place. Plan risk #7 calls this out: the GUI must locate the `tb` binary in PATH and fall back to a configurable absolute path when not found.

## Acceptance Criteria

- [ ] `gui/internal/cli/cli.go` exports a `Client` type with `Run(ctx, args...) ([]byte, error)` and `RunJSON(ctx, out any, args...) error`
- [ ] `Client` is constructed with a binary path; constructor uses `exec.LookPath("tb")` by default and accepts an override
- [ ] Non-zero exit codes return a typed error containing exit code + stderr (truncated for log safety)
- [ ] Stderr captured into the GUI's structured logger; not mixed into stdout/JSON
- [ ] `context.Context` cancellation kills the spawned process
- [ ] Unit tests cover: happy path JSON, non-zero exit, binary-not-found, context cancellation (use a sleep stub binary or `exec.Command("sh", "-c", "...")`)
- [ ] Public surface documented at the top of the file
