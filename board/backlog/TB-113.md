## Goal

Extend `tb start <ID>` so that, when worktree mode is enabled, it creates the per-task worktree (branch, exclude rules, sentinel, hooks, sidecar) and only then moves the task file from `backlog/` to `in-progress/`. The file move must be the last step so a failure mid-creation leaves the task in `backlog/`.

## Acceptance Criteria

- [ ] `tb start <ID>` follows the Â§ 4.5 algorithm exactly:
  1. Take `.worktree.lock(<ID>)` at `<board>/.worktree-locks/<ID>.lock`.
  2. Preflight (uses TB-111 checks): submodules, path conflict, branch collision, merge target exists.
  3. Resolve merge target SHA.
  4. Write sidecar `<board>/.worktrees/<ID>.json` with `state: "creating"` atomically.
  5. `git worktree add -b task/<ID> <worktrees.path>/<ID> <merge_target>`.
  6. Append `<board>/` and `.tb-worktree.json` to `$(git -C <wt> rev-parse --git-path info/exclude)`. Apply `--skip-worktree` to all tracked files under `<board>/`.
  7. Write sentinel `<wt>/.tb-worktree.json` (uses TB-112's writer).
  8. Install `pre-push` hook at `$(git -C <wt> rev-parse --git-path hooks/pre-push)` that rejects pushing any `<branch_prefix>*` ref.
  9. Update sidecar to `state: "active"`, `phase: "active"`, refreshed `baseSHA`.
  10. Take `.board.lock`; re-verify task is in `backlog/`; move file to `in-progress/`; append Log; regenerate `BOARD.md`; release `.board.lock`.
  11. Release `.worktree.lock`.
