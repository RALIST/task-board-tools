## Goal

Define `Runner`, `Mode`, `RunInput`, `RunResult`, and the embedded `prompts/implement.md`. The skeleton everything else in M4 plugs into.

## Acceptance Criteria

- [ ] `gui/internal/agent/runner.go` exports `Runner` interface — `Name() string` and `Run(ctx, RunInput) (RunResult, error)`
- [ ] `Mode` type with `ModeImplement` and `ModeGroom` constants; `String()` round-trips
- [ ] `RunInput{TaskID, Mode, Prompt, ProjectRoot, Env []string, Timeout time.Duration, Stdout, Stderr io.Writer, OnStarted func(pid, pgid int)}` — line streams are writers passed in, not returned from `RunResult`; `OnStarted` is the seam TB-47 uses to populate its `activeRun` map after `cmd.Start()` succeeds
- [ ] `RunResult{ExitCode int, Err error}` — `Err` non-nil for spawn/IO failures only; a non-zero `ExitCode` is not an error
- [ ] `gui/internal/agent/prompts/implement.md` embedded via `//go:embed`, exported as `PromptImplement string`
- [ ] **Prompt placeholders are fixed and documented in the file**: `{{TASK_ID}}`, `{{TASK_TITLE}}`, `{{TASK_BODY}}`. Exact tokens — no `{ }` variants, no whitespace inside braces
- [ ] `RenderPrompt(template string, vars PromptVars) string` substitutes the three placeholders by exact-string replace (no regex, no template engine). `PromptVars{TaskID, TaskTitle, TaskBody string}`
- [ ] Unit tests: `PromptImplement` is non-empty AND contains all three placeholder tokens verbatim; `RenderPrompt` replaces every occurrence; unknown placeholders in the template pass through unchanged; `Mode` constants render correctly
