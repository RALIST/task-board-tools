# TB-39: addChildToSubtasks corrupts task body when '## ' headers appear inside backticks

**Type:** bug
**Priority:** P2
**Size:** S
**Module:** cli
**Tags:** robustness,parser,quick-win
**Branch:** —

## Goal

tb create with --parent injects a ## Subtasks section by searching for the first '## Acceptance Criteria' line. If a parent task's body contains that string inside a code span (e.g. context paragraph quoting section names), the injection lands mid-paragraph and corrupts the body.

## Acceptance Criteria

- [ ] `addChildToSubtasks` in `cli/create.go` only matches `## ` headers when the marker is at column 0 of a line that is structurally a heading — not when it appears inside an inline code span (backticks) or a fenced code block
- [ ] Repro test: create an epic whose `## Context` body contains the literal `` `## Acceptance Criteria` `` inside backticks; create a child with `--parent`; assert the parent body is unchanged except for the new `## Subtasks` block inserted before the real heading
- [ ] The new `## Subtasks` block is inserted before the real `## Acceptance Criteria` heading (or appended at end if no such heading exists), atomically, under `.board.lock`
- [ ] Same fix audited for any other call sites that scan for section headings (grep for `"## "` in `cli/*.go`)
