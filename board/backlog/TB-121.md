# TB-121: tb doctor: surface unresolvable worktree state

**Type:** feature
**Priority:** P1
**Size:** S
**Module:** cli
**Branch:** —
**Parent:** TB-109

## Goal

New `tb doctor` command. Diagnoses and reports worktree-mode state that the § 4.12 reconciliation table couldn't safely auto-fix — orphan worktrees, orphan branches, corrupt sidecars, inconsistent state-and-taskDir rows, mid-rebase recovery cases. Output is human-readable; supports `--json` for tooling.

## Context

Spec sections: § 4.12 (recovery scans + reconciliation table), § 4.13 (corrupt sidecar handling).

Recovery (TB-117) was specced to "log to tb doctor" for every unresolvable case but `tb doctor` itself doesn't exist yet. Without this command the recovery ACs (AC8c, AC8g, AC8i, and the "log to tb doctor" rows of the reconciliation table) have no actual surface to assert against.

## Acceptance Criteria

- [ ] New CLI subcommand `tb doctor` registered in `cli/main.go`.
- [ ] When run, `tb doctor` performs the same two scans as § 4.12 recovery (Scan A — sidecars; Scan B — `git worktree list`) but in **read-only** mode (no reconciliation actions). Reports every row that the reconciliation table marks as "log to tb doctor".
- [ ] Reports include: orphan worktrees (in `git worktree list` but no sidecar), worktree-missing-but-sidecar-present in `active/active/in-progress`, all inconsistent `state/phase/taskDir` combinations, corrupt sidecars (unreadable JSON), unresolvable `finalizing` cases (FF didn't happen but taskDir is `done`/`archive`/etc.).
- [ ] Output formats:
  - Default: human-readable summary, exit code 0 if clean, 1 if any issues found.
  - `--json`: structured array of `{id, kind, severity, message, path?}` entries.
- [ ] Each finding includes the recommended remediation command (e.g. `tb worktree clean <ID> --force`, `manual: resolve <wt>/path then tb merge <ID>`).
- [ ] Recovery (TB-117) emits via a shared helper so `tb doctor` and startup recovery agree on findings.
- [ ] Integration test: seed each kind of inconsistency, assert `tb doctor --json` reports it.
