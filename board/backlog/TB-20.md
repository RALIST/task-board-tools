# TB-20: fsnotify watcher with debounce and Wails events

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** gui

**Branch:** —
**Parent:** TB-2

## Goal

Implement gui/internal/watcher/watcher.go using fsnotify. Watches board/{backlog,in-progress,done,archive}/. Ignores BOARD.md, .next-id, .board.lock, .agent-state/, .agent-logs/. Debounce 200ms. Emits Wails events board:reloaded on Create/Remove/Rename and task:updated:<id> on Write.

## Context

Live update path for the GUI. Every CLI mutation eventually writes a task `.md` and then a fresh `BOARD.md`. The watcher translates filesystem activity into two coarse-grained Wails events: a column-level invalidation (`board:reloaded`) and a single-task invalidation (`task:updated:<id>`). Debounce coalesces the fan-out from a single logical mutation (move = remove + create + regenerate). Plan risk #2 explicitly calls out the ignore list — emitting on `BOARD.md` would create a feedback loop because `tb regenerate` rewrites it after every mutation. The watcher depends on atomic writes (TB-9/TB-10) so it never sees a partially written file.

## Acceptance Criteria

- [ ] `gui/internal/watcher/watcher.go` exports `Watcher` with `Start(ctx, boardDir)` and a `Switch(boardDir)` method for `OpenBoard`
- [ ] Subscribes to `board/{backlog,in-progress,done,archive}/`
- [ ] Ignores writes to `BOARD.md`, `.next-id`, `.board.lock`, anything under `.agent-state/`, anything under `.agent-logs/`
- [ ] 200ms debounce window — multi-event bursts (remove + create + regenerate) emit a single `board:reloaded`
- [ ] `Create`, `Remove`, `Rename` events emit `board:reloaded`
- [ ] `Write` events on a task `.md` emit `task:updated:<id>` (id derived from filename)
- [ ] Graceful shutdown on `ctx.Done()` — fsnotify watcher closed, goroutines drained
- [ ] Integration test: drive `tb mv` against a temp board, assert exactly one `board:reloaded` is received within 1s
