# TB-84: Keyboard shortcuts: N (new), / (search), Esc (close drawer), Enter (open card)

**Type:** feature
**Priority:** P2
**Size:** S
**Module:** gui
**Tags:** milestone-m7,gui,frontend
**Branch:** —
**Parent:** TB-7

## Goal

Wire global keyboard shortcuts in the kanban view:
- `N` — open `CreateTaskDialog`
- `/` — focus the search/filter input in `FilterBar`
- `Esc` — close the active drawer or dialog
- `Enter` — open the currently-selected card's drawer

Plus card selection (arrow keys to move between cards) so `Enter` has a target.

## Context

Per `docs/FEATURES.md` F7.2. Today only `Esc` is wired (in `TaskDrawer.svelte`).

**Conflict-aware handling**: bare-letter shortcuts (`N`, `/`) must be suppressed inside form inputs (textarea, contenteditable, CodeMirror) so typing in the body editor or the create-dialog title field doesn't open another modal. Use a single top-level handler in `+page.svelte` that:
1. Checks `event.target.matches('input, textarea, [contenteditable]')` and bails for letter shortcuts.
2. Checks `event.target.closest('.cm-editor')` to bail inside CodeMirror.
3. Checks if any modal/drawer is open and bails for `N` / `/` so the user can't pile dialogs.
4. `Esc` closes the topmost modal/drawer; if none is open, clears card selection.

**Card-focus state** — `gui/frontend/src/lib/stores/selection.ts` today only tracks the open-drawer task ID (`selectedTaskId: writable<string | null>`); it has no card-focus or keyboard-navigation slot. Two options for the implementer:
- (a) extend `selection.ts` with a `focusedTaskId` writable + helpers (`focusNext`, `focusPrev`, `focusColumn(dir)`) so `Enter` reads the focused id and calls `openTask`.
- (b) skip the store change and rely on native DOM focus — make each `Card.svelte` `tabindex="0"` so `Enter` activates the focused card via a keydown handler on the card itself, falling out of the global handler's scope.

(b) is the smaller change and works for an S-sized task. (a) is required if column-jump arrow-key navigation is in scope. Pick (b) for M7 and document arrow-key column traversal as a deferred polish item; AC below reflects (b).

## Acceptance Criteria

- [ ] `N` opens `CreateTaskDialog` (only when no modal/drawer is already open and focus is not in an input)
- [ ] `/` focuses the FilterBar search input
- [ ] `Esc` closes whichever surface is topmost (modal > drawer); when none open, blurs any focused card
- [ ] `Enter` opens the drawer for the DOM-focused card (`Card.svelte` gets `tabindex="0"` + a keydown handler); arrow-key column traversal is out of scope and noted in the parent epic for a post-M7 polish task
- [ ] Bare-letter shortcuts suppressed inside `<input>`, `<textarea>`, `[contenteditable]`, and `.cm-editor`
- [ ] Settings panel (TB-81) suppresses `N` and `/` while open (validated together with TB-81)
- [ ] Vitest test coverage for the input-suppression branch and the topmost-modal branch
- [ ] No new dependency (no Mousetrap / hotkeys-js — vanilla `keydown` handler is enough for 4 shortcuts)
