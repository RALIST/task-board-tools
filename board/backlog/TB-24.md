## Goal

Implement TaskDrawer.svelte as a right-side slide-over. Opens when a card is clicked. Shows metadata fields (read-only display) and renders the markdown body via svelte-markdown. No editing in M2 — CodeMirror editor lands in M3.

## Context

The detail view in M2 is read-only. Body rendering uses `svelte-markdown` (the editor lands in M3). Drawer fetches a single task via `BoardService.GetTask` and subscribes to `task:updated:<id>` so external CLI edits propagate live while the drawer is open. CLAUDE.md mandates using the `frontend-design` skill — invoke it before writing the component.

## Acceptance Criteria

- [ ] `TaskDrawer.svelte` is a right-side slide-over (overlay, not push) with `Esc` to close and a click-outside trap
- [ ] Header: task ID + title; Body: metadata grid (Type, Priority, Size, Module, Tags, Branch, Parent, Agent, AgentStatus) + rendered markdown
- [ ] Calls `api.getTask(id)` when opened; shows a loading state while in flight
- [ ] Subscribes to `task:updated:<id>` while open and re-fetches on event
- [ ] Closes cleanly and unsubscribes when navigation away
- [ ] Renders sanely for tasks missing optional fields (Module, Tags, Parent, Agent) — no `undefined` leaks
- [ ] No mutation controls — every input field is display-only (mutations land in M3)
- [ ] Implementer follows the `frontend-design` skill for visual quality (per CLAUDE.md instruction)
