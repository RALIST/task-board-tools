# TB-12: Add --json output to ls, show, and board

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** cli
**Tags:** milestone-m1,json
**Branch:** —
**Parent:** TB-1

## Goal

Add a --json flag to tb ls, tb show, and tb board (cmdBoard in regenerate.go). All Task fields serialized camelCase. Empty results return [] or {} (never prose like 'No tasks found.'). Stdout = data, stderr = errors/warnings. New cli/json_output.go holds the serializers.

## Context

GUI needs structured output to avoid parsing tabwriter. Use `encoding/json` from stdlib (still zero-dep). Stdout = data, stderr = errors/warnings only. Empty results must produce `[]` or `{}` so `jq` pipelines don't break.

## Acceptance Criteria

- [ ] `cli/json_output.go` exports serializers for `Task` and `BoardSnapshot`
- [ ] All `Task` fields camelCase: `id, title, type, priority, size, module, tags, branch, parent, status, filePath, agent, agentStatus`
- [ ] `tb ls --json` returns `[]` for an empty selection (never prose)
- [ ] `tb show <ID> --json` returns `{metadata, body}` where `metadata` is the Task struct and `body` is the full markdown
- [ ] `tb board --json` returns `{epics, activeEpics, finishedEpics, inProgress, backlog, recentlyDone}` (all arrays default to `[]`)
- [ ] `jq .` parses every output without error
- [ ] Stdout-only: errors and warnings go to stderr
