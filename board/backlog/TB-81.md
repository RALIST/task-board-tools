## Goal

Add a settings panel — slide-over or modal — that surfaces the four preferences (`agent_timeout_minutes`, `max_workers`, `default_agent`, `cli_path`) with sensible inputs, client-side validation hints matching the backend clamps, a Save button that calls `preferencesStore` setters, and a toast on success/failure.

## Context

Implements `docs/FEATURES.md` F7.1. The panel is opened from the application menu (TB-82) and via a settings glyph in the topbar (Cmd+,). Use existing patterns:
- Layout: lean on the same slide-over scaffold `TaskDrawer.svelte` uses (Esc to close, click-outside to dismiss) so the GUI feels consistent.
- Inputs: number input for timeout (1–240), number input for max workers (1–4), select for default_agent (`none|claude|codex`), text input + "Browse…" button for cli_path (uses `PickBoardDialog`-style picker with `CanChooseFiles(true)` if the Wails3 alpha API supports a file picker).
- Validation: client side mirrors backend clamps so the user sees the number snap; backend remains the source of truth.
- A single Save button persists all four fields in one go, but each setter is its own atomic write — partial failure surfaces a per-field error in the toast.

Use the `frontend-design` skill for visual polish. Avoid adding heavyweight UI deps (no Tailwind, no Material kit) — match the existing CSS-in-Svelte style.

## Acceptance Criteria

- [ ] `SettingsPanel.svelte` renders all four fields with current values from `preferencesStore`
- [ ] Save button is disabled while no field is dirty; on click it calls the relevant `set<Field>` methods and shows a success toast (or per-field error toast on failure)
- [ ] Number inputs enforce min/max + step; out-of-range entries snap to bounds on blur (matching backend clamps from TB-76)
- [ ] `cli_path` input has a "Browse…" button that opens a file picker and writes the chosen path
- [ ] Esc closes the panel; click-outside closes it (same UX as `TaskDrawer`)
- [ ] No regression in keyboard shortcuts — `N` and `/` are suppressed while the panel is open
- [ ] Smoke (`/ui-test` at end of M7): change agent timeout → run an agent → run terminates with new timeout (covers TB-77 wiring end-to-end)
