# TB-21: SettingsService: project root, recent boards, folder picker

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** gui
**Tags:** milestone-m2,wails-service,settings
**Branch:** —
**Parent:** TB-2

## Goal

Implement gui/app/settings_service.go with GetProjectRoot, GetBoardInfo, OpenBoard(projectRoot), PickBoardDialog (native Wails3 dialog), and ListRecentBoards (read/write ~/.config/tb-gui/recent.json). On first launch, prompt the folder picker and remember. OpenBoard reloads config, redirects watcher, and emits board:reloaded.

## Context

GUI needs to know which board it is showing. The plan settles this clearly (risk #4): `OpenBoard` accepts the project root (the directory containing `.tb.yaml`), not the board directory itself, so the existing `loadProjectConfig` walk works the same way the CLI does. The recent-boards file is small persistent state — keep it in the OS config dir, not inside any board.

## Acceptance Criteria

- [ ] `gui/app/settings_service.go` declares `SettingsService` with: `GetProjectRoot()`, `GetBoardInfo()`, `OpenBoard(projectRoot string)`, `PickBoardDialog() (string, error)`, `ListRecentBoards() ([]RecentBoard, error)`
- [ ] `BoardInfo` struct returns `{projectRoot, boardDir, prefix, wipLimit}` parsed from `.tb.yaml`
- [ ] `OpenBoard(projectRoot)` validates `.tb.yaml` exists, swaps active config, calls `watcher.Switch`, prepends the entry to the recent list, then emits `board:reloaded`
- [ ] `PickBoardDialog` uses the native Wails3 dialogs API
- [ ] `RecentBoard` shape: `{projectRoot, lastOpened, prefix}`. Stored as JSON at `$XDG_CONFIG_HOME/tb-gui/recent.json` (fallback `~/.config/tb-gui/recent.json` on macOS/Linux)
- [ ] Recent list deduplicated by `projectRoot`, capped to 20 entries
- [ ] On app start, if no recent boards or the last one no longer has `.tb.yaml`, the dialog opens automatically
- [ ] Opening a path without `.tb.yaml` returns a typed error the frontend can present as a toast
- [ ] Service registered with Wails so it shows up in frontend bindings
