# TB-112: Board path resolution from inside a worktree

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** cli
**Branch:** —
**Parent:** TB-109

## Goal

Make `tb` invocations from inside a worktree always resolve to the canonical board, never to the worktree-local snapshot. Implement the three-layer resolution (env var > sentinel > .tb.yaml) and the sentinel writer used by worktree creation.

## Context

Spec sections: § 3.1 (resolution sentinel), § 11 decision 10.

Without this, a human running `cd .tb-worktrees/PR-N && tb edit PR-N` would silently mutate the worktree-local copy of `board/`, which is exactly the divergence the canonical-board invariant exists to prevent.

## Acceptance Criteria

- [ ] `tb`'s board-path resolution algorithm is updated to:
  1. `TB_BOARD_PATH` env var (highest priority).
  2. Walking up from cwd, first match wins: `.tb-worktree.json` (use `canonical_board` field), OR `.tb.yaml` (use its configured board path).
  3. Error if nothing found.
- [ ] `.tb-worktree.json` writer: a helper that, given a worktree path and the canonical board's absolute path, writes the sentinel atomically (`writeFileAtomic`) at `<wt>/.tb-worktree.json` with `{ "canonical_board": "...", "task_id": "TB-NNN" }`.
- [ ] AC2 of the spec: from inside a worktree, with `unset TB_BOARD_PATH`, `tb edit <ID> -d "x"` mutates the canonical board (visible from main checkout) and leaves the worktree's `board/` unmodified.
- [ ] Unit test: resolution has cases for env-var-wins, sentinel-wins-over-yaml, sentinel-absent-yaml-wins, none-found-errors.
