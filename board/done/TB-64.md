# TB-64: GUI "Open project" button only works once, subsequent clicks do nothing

**Type:** bug
**Priority:** P1
**Size:** S
**Module:** gui
**Tags:** gui

## Goal

Make the "Open project" button in the GUI work on every click, not just the first one.

## Context

First click on "Open project" successfully invokes `pickBoardDialog()` and switches the active board via `openBoard()`. On every subsequent click nothing happens — no dialog appears and no error is logged.

Suspected causes (to confirm during fix):

- A "picker open" guard flag is set true on first click and never reset (e.g. early-return on second invocation).
- The Wails3 native `OpenFile` dialog promise from the first call hasn't been fully released and a new invocation is silently rejected.
- An exception in `openBoard` after the first success puts the page into a state where the button handler is no longer wired (event listener detached on store reload).

Call sites: `gui/frontend/src/routes/+page.svelte:95` (`pickBoardDialog()` then `openBoard(path)`), backed by `gui/frontend/src/lib/api.ts:120-124` calling generated bindings `OpenBoard` / `PickBoardDialog` on `SettingsService`.

## Acceptance Criteria

- [x] Clicking "Open project" opens the native folder picker every time, regardless of how many times it was clicked before
- [x] Cancelling the picker (no selection / ErrCancelled) leaves the current board active and does NOT block subsequent clicks
- [x] Successfully picking the same project twice in a row re-opens it without error
- [x] Picking a path without `.tb.yaml` surfaces the existing `ErrNoTbYaml` toast and does NOT block subsequent clicks
- [x] If the bug was caused by a guard flag / pending-promise state, the regression is covered by either a unit/component test or a clearly-named comment + manual repro recorded in the Log
- [x] `npm run check` → 0 errors / 0 warnings
