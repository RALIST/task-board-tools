## Goal

Add tb attach to copy/import files into <status>/<ID>/attachments/ under .board.lock; when invoked on a legacy file task, atomically promote it to folder form while preserving metadata, log history, and BOARD.md rendering.

## Acceptance Criteria

- [x] `tb attach <ID> <path>...` copies / imports files into `<status>/<ID>/attachments/` under `.board.lock`; name collisions fail before publishing new attachments, as documented in `cli/README.md`.
- [x] If the target is a legacy file-form task, the command atomically promotes it to folder form, preserving metadata, log history, and `BOARD.md` rendering; promotion and attach entries are appended to the task Log.
- [x] Failure paths (partial copy, missing source, lock contention) leave the board in a consistent state with no half-copied attachments.
- [x] `cli/` tests cover file-form target, folder-form target, multi-file, and failure cases.
