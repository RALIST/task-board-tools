# TB-63: GUI tag header overflows when too many tags — collapse to popular + dropdown

**Type:** bug
**Priority:** P1
**Size:** M
**Module:** gui
**Tags:** gui,ui,quick-win
**Branch:** —

## Goal

Cap the FilterBar tag row so it can't push the board offscreen when the board has many tags. Show the most-used (or most-recently-selected) tags inline and fold the rest into an overflow dropdown.

## Context

`FilterBar.svelte:87` renders `tags.slice(0, 16)` as a flat row of chips. On boards with many tags the 16 chips still wrap across multiple lines and consume most of the viewport vertical space; on narrow windows even fewer can fit before the row eats the page. We need a hard cap on visible chips with a "+N more" affordance.

## Acceptance Criteria

- [x] Tag row never exceeds a single line at the configured minimum window width (720px)
- [x] Visible inline tags are ranked by frequency in the current snapshot (popular first); active filter selections always remain visible inline regardless of rank
- [x] Overflow tags collapse into a "+N more" dropdown/popover that lists the remainder with the same toggle behaviour
- [x] Dropdown is keyboard-accessible (open on Enter/Space, navigate with arrows, close on Esc/blur)
- [x] Selecting a tag from the dropdown reflects its active state in the bar (chip moves inline or shows a count badge)
- [x] No regressions to the existing `toggle('tags', v)` logic — selecting still patches `f.tags` reactively
- [x] `npm run check` → 0 errors / 0 warnings
