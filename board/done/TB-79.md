## Goal

When the user opens the Agent dropdown in `TaskDrawer.svelte` for a task without `**Agent:**` set, pre-select the value of the `default_agent` preference (TB-76) instead of the literal `none`. The pre-selection is **visual only** â€” `formAgent` stays in sync with the displayed default but `onAgentChange` (`TaskDrawer.svelte:155`) never fires until the user actively changes the selection, so a user who never opens the dropdown never accidentally writes the agent metadata.

## Context

Per TB-7's groomed Goal: the GUI does not auto-assign on `tb create`. Assignment remains a deliberate user gesture. The current drawer (`TaskDrawer.svelte:101`) initialises `formAgent` from `detail.metadata.agent ?? ''` and `onAgentChange` (`:155`) auto-saves on `<select>` change â€” there is no explicit Assign button. M7's tweak is purely cosmetic: when `detail.metadata.agent` is empty, surface `default_agent` as the dropdown's *displayed* option without staging an assignment.

The implementation must avoid a footgun: `onAgentChange` triggers when the bound value changes. If we simply set `formAgent = defaultAgent` on hydration, the existing `target === prev || (target === 'none' && prev === '')` guard at `TaskDrawer.svelte:160` will not catch `claude !== ''` and will auto-save. Two clean options:
- (a) keep `formAgent` empty internally; render the `<select>` with a `value` override that points at `defaultAgent` while `formAgent` is empty, and only mutate `formAgent` from `on:change` (skips the auto-save trigger).
- (b) extend the equality guard to short-circuit when the change is "from `''` to whatever default_agent is and the user hasn't interacted yet" â€” flagged by an `interacted` boolean set in `on:change`.

(a) is cleaner and doesn't introduce hidden state in the auto-save path. Pick (a) unless it conflicts with the dropdown's a11y semantics.

If the persisted `default_agent` is `none` or unset, the dropdown's default stays at `none` (current behaviour).
