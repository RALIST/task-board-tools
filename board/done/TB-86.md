## Goal

Add a Codex-specific post-edit hook under `.codex/` that understands Codex PostToolUse payloads and returns supported JSON feedback without changing the Claude hook.

## Acceptance Criteria

- [x] Hook lives under `.codex/` and `.claude/hooks/post-edit.sh` is left unchanged.
- [x] Hook derives edited paths from Codex `PostToolUse` payloads for `apply_patch` and from direct file-path payloads when present.
- [x] Hook uses Codex-supported JSON feedback (`systemMessage`, `decision`, `reason`, `hookSpecificOutput.additionalContext`, `continue`) and avoids relying on unsupported output fields.
- [x] Codex hook matcher uses the canonical `apply_patch` tool name.
- [x] Shell syntax and representative payload handling are verified locally.
- [x] Go hook path formats edited files with `gofmt` before `go vet` and uses a temp `GOCACHE` that works under the sandbox.
