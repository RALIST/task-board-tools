# TB-88: GUI triage fails when active tb binary lacks JSON support

**Type:** bug
**Priority:** P2
**Size:** M
**Module:** gui
**Tags:** gui,triage,cli-path,settings,robustness
**Branch:** —

## Goal

The GUI should not surface a raw Wails binding failure when `BoardService.Triage()` calls an older or stale `tb` binary whose `triage --json` command exits successfully but prints the human `Found ... task(s) needing grooming` output.

## Context

The current source implementation (`cd cli && go run . triage --json`) emits a JSON array, but the installed binary on PATH can still be stale. In this checkout, `/Users/ralist/go/bin/tb triage --json` prints the tabwriter view beginning with `Found`, matching the reported Wails error:

```text
1:05PM ERR Binding call failed: Bound method returned an error: tb [triage --json]: decode JSON: invalid character 'F' looking for beginning of value
```

`gui/app/board_service.go` currently calls `c.RunJSON(ctx, &rows, "triage", "--json")` and propagates the JSON decode error. Triage badges are advisory, so a stale triage-capability failure should not break the binding path or board rendering.

## Acceptance Criteria

- [x] `BoardService.Triage()` handles invalid JSON from `tb triage --json` as a triage-unavailable condition instead of returning the raw JSON decode error to Wails
- [x] The handled path logs or otherwise records an actionable diagnostic that includes the active `tb` binary path and explains that the selected CLI must support `triage --json`
- [x] The frontend receives a resolved empty triage map for this condition, so cards render without grooming badges and no `Binding call failed` log is produced for advisory triage refreshes
- [x] Other CLI failures still propagate normally: missing `tb`, non-zero exit, no board selected, and context cancellation must keep their current error behavior
- [x] Add a `gui/app` regression test with a stub `tb` that exits 0 and prints `Found ...` for `triage --json`; assert `Triage()` resolves to an empty map and does not return the JSON decode error
- [x] Keep the happy-path `tb triage --json` cache/invalidation behavior from TB-70 unchanged
- [x] Verify with `cd gui && go test ./...`
