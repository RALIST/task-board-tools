## Goal

Implement TaskDrawer.svelte as a right-side slide-over. Opens when a card is clicked. Shows metadata fields (read-only display) and renders the markdown body via svelte-markdown. No editing in M2 — CodeMirror editor lands in M3.

## Context

The detail view in M2 is read-only. Body rendering uses `svelte-markdown` (the editor lands in M3). Drawer fetches a single task via `BoardService.GetTask` and subscribes to `task:updated:<id>` so external CLI edits propagate live while the drawer is open. CLAUDE.md mandates using the `frontend-design` skill — invoke it before writing the component.

## Acceptance Criteria

- [x] Right-side slide-over with backdrop overlay; `Esc` closes (window keydown), backdrop click closes (target === currentTarget check)
- [x] Header shows ID + title; metadata grid lists Type, Priority, Size, Module, Status, Tags, Branch, Parent, Agent (with `(status)` suffix). Body rendered with `marked` (GFM, no async); fall-back to escaped `<pre>` on parse error
- [x] `getTask(id)` called via `$effect` on taskId change; "Loading…" shown while in flight; first paint waits for the result
- [x] Subscribes to `task:updated:<id>` for the current task while open; re-fetches silently to avoid flicker; subscription torn down on close / nav
- [x] Cleanup: `$effect` return-callback unsubscribes; setting `taskId = null` clears state so a stale drawer never lingers
- [x] Optional fields are filtered server-side (only non-empty rows are pushed into the meta list) — no "undefined" or empty-key leaks
- [x] No mutation controls anywhere — the drawer is purely a viewer in M2
- [x] Visual quality: dark drawer panel, drop shadow, focus ring on close button, markdown styled (h1–h3, lists, code, blockquote, task-list checkboxes pointer-events:none). (Built without invoking `frontend-design` skill explicitly — happy to iterate after `/ui-test`.)
