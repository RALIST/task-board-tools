## Goal

Promote the inline toast markup from +page.svelte into a reusable Toast.svelte component bound to the existing toast store. All mutation services surface failures through it.

## Acceptance Criteria

- [ ] `gui/frontend/src/lib/components/Toast.svelte` subscribes to the existing `toasts` store and renders the toast stack
- [ ] Visual parity (or better) with the inline implementation currently in `+page.svelte` — dismiss-on-click, auto-dismiss after 5s, `kind` styling (error/info/success)
- [ ] `+page.svelte` replaces its inline toast markup with `<Toast />`; no duplicate rendering
- [ ] The component is mounted once at the app shell so every mutation (TB-32/33/34/35/36/37) gets consistent error feedback
- [ ] Keyboard-accessible: dismiss button is focusable
- [ ] Closes one item on `docs/IMPLEMENTATION.md` M3 task list (Frontend `Toast.svelte` for errors)
