## Goal

Set up the SvelteKit shell under gui/frontend: typed Wails bindings in lib/api.ts, board+filter stores in lib/stores/, and the single route +page.svelte that subscribes to board/task events and re-fetches via api.LoadBoard. Empty visual state — components land in TB-23.

## Context

Wires the frontend to the Go services so subsequent component work (TB-23, TB-24) only deals with rendering. `api.ts` is a thin typed layer over the Wails-generated bindings so the rest of the frontend never imports Wails directly — that lets us swap in a mock for storybook/tests later. Stores wrap the snapshot and merge per-task updates from `task:updated:<id>` events without re-fetching the whole board.

## Acceptance Criteria

- [x] `gui/frontend/src/lib/api.ts` exposes typed `loadBoard`, `getTask`, `openBoard`, `pickBoardDialog`, `listRecentBoards`, `getProjectRoot`, `getBoardInfo` wrapping the Wails bindings; also `isNoTbYamlError` / `isCancelledError` helpers for error-branch styling
- [x] `gui/frontend/src/lib/stores/board.ts` — Svelte store; `task:updated:<id>` events call `patchTask(id)` which re-fetches a single task and splices it into the right column (no full reload)
- [x] `gui/frontend/src/lib/stores/filter.ts` — placeholder shape `{search, showArchive}` ready for M3 FilterBar
- [x] `gui/frontend/src/routes/+page.svelte`: on mount probes `getProjectRoot`, falls back to most-recent in `recents`, falls back to picker; subscribes to `board:reloaded`, `board:opened`, `task:updated:*`; renders columns via `<Board>` (TB-23)
- [x] Empty-state UI when no board is selected — call to action button + recent-board list both trigger the picker / openBoard
- [x] Toast surface via `stores/toast.ts` + in-page `<aside class="toasts">`; 5s auto-dismiss; click-to-dismiss
- [ ] `wails3 dev` renders the shell against the live board with zero console errors — runtime test deferred to end-of-epic `/ui-test`
