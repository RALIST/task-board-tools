# TB-40: BoardService.LoadBoard: archive-aware status mode

**Type:** feature
**Priority:** P1
**Size:** S
**Module:** gui
**Tags:** milestone-m3,wails-service,archive
**Branch:** —
**Parent:** TB-3

## Goal

Extend BoardService.LoadBoard and BoardSnapshot so the Archive column can render. LoadBoard accepts a status mode ('active' | 'all'); BoardSnapshot gains an 'archive' bucket; the CLI exec passes '--status all' when requested.

## Acceptance Criteria

- [ ] `BoardService.LoadBoard` accepts a status mode parameter (`active` default, `all`); existing callers default to `active` so M2 behavior is preserved
- [ ] `BoardSnapshot` gains an `archive` bucket alongside `backlog`, `inProgress`, `done`
- [ ] `gui/internal/cli.Board` passes `--status all` when mode is `all`
- [ ] Wails bindings regenerated; `api.ts` exposes the new parameter
- [ ] Frontend `boardStore` adds `archive` and a status-mode setter; existing kanban unchanged when mode is `active`
- [ ] Unit tests cover both modes against a fake `tb` binary
- [ ] Prerequisite for TB-38 (FilterBar's "Show archived" toggle)
