# TB-44: ClaudeRunner and CodexRunner: exec.CommandContext with own process group

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** gui
**Tags:** milestone-m4,agent,runner
**Branch:** —
**Parent:** TB-4

## Goal

Concrete `Runner` implementations that shell out to `claude -p <prompt>` and `codex exec --prompt <prompt>`. Own process group so kill cascades to children; env whitelist so the agent doesn't see ambient secrets; line-scanned stdout/stderr forwarded to writers in `RunInput`. Cancellation **mechanism** lives here (process group + `OnStarted` callback exposing PID/pgid); cancellation **policy** (SIGTERM, 5s grace, SIGKILL) is TB-48's.

## Acceptance Criteria

- [ ] `ClaudeRunner.Run` invokes `claude -p <prompt>` (prompt as positional arg after `-p`)
- [ ] `CodexRunner.Run` invokes `codex exec --prompt <prompt>` (prompt via flag — confirmed at implementation time against `codex --help` and recorded in a source comment with the exact `codex --version` tested). If `codex exec` rejects `--prompt`, fall back to `codex exec` with prompt on stdin; either path must be explicitly tested. Resolves the R-CLI-shape risk in `docs/IMPLEMENTATION.md` M4 risks
- [ ] `exec.Command(name, args...)` (not `exec.CommandContext` — see below) with `Dir = RunInput.ProjectRoot`
- [ ] `Env` restricted to a whitelist: `HOME`, `PATH`, `USER`, `LANG`, `LC_*`, `TERM`, plus anything the caller passes in `RunInput.Env`. Ambient env (e.g. `AWS_*`, `OPENAI_API_KEY` from other shells) is **not** forwarded by default
- [ ] `cmd.SysProcAttr = &syscall.SysProcAttr{Setpgid: true}` so `syscall.Kill(-pgid, SIGKILL)` reaches the agent's children
- [ ] After `cmd.Start()` succeeds, call `RunInput.OnStarted(cmd.Process.Pid, pgid)` synchronously so TB-47 can populate its `activeRun` map BEFORE the runner blocks on output streaming. `pgid` is the leader's PID (matches Pid because of Setpgid)
- [ ] Stdout/stderr captured via `cmd.StdoutPipe`/`StderrPipe`, scanned by `bufio.Scanner` (with `Buffer(make([]byte, 0, 64*1024), 1<<20)` for long lines), forwarded line-by-line to `RunInput.Stdout`/`Stderr`
- [ ] Context cancellation: a goroutine waits on `ctx.Done()` and calls `cmd.Process.Signal(syscall.SIGTERM)` once. The runner does NOT escalate to SIGKILL — TB-48 owns that escalation by reaching into `activeRun.Pgid` directly. If `ctx` is the deadline-style timeout (`RunInput.Timeout`), TB-44 also sends SIGTERM and then SIGKILL to the pgid after 5s (this is the unattended-timeout path, distinct from user cancel)
- [ ] On `cmd.Wait` return: `RunResult{ExitCode, Err}`. `ExitCode` = `cmd.ProcessState.ExitCode()`. `Err` is non-nil only for: spawn failure (`ErrBinaryNotFound` mapped from `*exec.Error`), IO error reading pipes, or unattended timeout (`ErrTimeout`). Context-cancelled (user cancel) returns `Err = ctx.Err()` and `ExitCode = -1`
- [ ] Unit tests against a tiny stub binary built at test time (mirrors `gui/internal/watcher/integration_test.go` style): (a) line ordering preserved across stdout+stderr, (b) exit code captured (0 and non-zero), (c) `ctx` cancellation delivers SIGTERM and `Run` returns within ~1s with `Err = context.Canceled`, (d) timeout path delivers SIGTERM then SIGKILL within 5s and child processes spawned by the stub also die (PID-liveness probe), (e) `OnStarted` fires before the first stdout line is forwarded
