# TB-50: TaskDrawer agent dropdown + Run/Cancel buttons + Card agent badge

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** gui
**Tags:** milestone-m4,frontend,drawer,card
**Branch:** —
**Parent:** TB-4

## Goal

The visible surface for M4. Drawer gets the controls; Card.svelte gets a small badge so agent-assigned tasks are spottable in the column at a glance.

## Acceptance Criteria

- [ ] `TaskDrawer.svelte` adds an "Agent" `<select>` bound to `currentTask.agent`; on change → `AgentService.AssignAgent`; pushes a `Toast` on error
- [ ] **Run** button visible when an agent is assigned; enabled iff `AgentStatus ∉ {queued, running}`; click → `AgentService.RunAgent`; the returned `run_id` is forwarded to the local `selectedRunID` so `AgentRunLog` starts streaming
- [ ] **Cancel** button visible iff `AgentStatus === 'running'`; uses the same two-click confirm pattern as the existing Archive button; click → `AgentService.CancelRun`
- [ ] Status pill (`queued | running | success | failed | cancelled`) rendered near the buttons, sourced from the selected `Run` in `runsStore` (not `currentTask.agentStatus`, which lags by one `tb edit`)
- [ ] `Card.svelte` shows a small agent badge (icon + first letter — `C` for claude, `X` for codex) next to the existing priority/type pills when `task.agent` is set; hidden when none
- [ ] Disabled-button cursor and tooltip explain why (e.g. "Assign an agent first", "Already running")
- [ ] **Frontend-only verification** (Vitest + Testing Library): mounting the drawer with various `(agent, agentStatus)` combinations renders the right buttons in the right enabled/visible state; clicking each button calls the right `AgentService` method with the right args (mocked binding). End-to-end on-disk behavior is covered by TB-46 (assignment persistence), TB-47 (run lifecycle), TB-48 (cancel lifecycle) — TB-50 does not duplicate those
