## Goal

Make CLI status moves, close/archive, and restore preserve folder-form tasks as one complete task directory (`<status>/<ID>/TASK.md` plus task-local artifacts) without leaving orphaned files behind.

## Context

This is the CLI move/archive slice of parent epic TB-93. The relevant implementation surfaces are @cli/move.go (`tb mv`, `tb start`, `tb done`, shared move flow), @cli/board.go (`tb close` / archive behavior and task lookup), @cli/regenerate.go (`BOARD.md` rebuild), and the folder-task contract that TB-94 is expected to define in @docs/ARCHITECTURE.md.

Today the CLI move/archive code is file-oriented. After TB-96/TB-97 introduce folder-form task reads/creation, this task makes moving a folder task behave like moving the task itself rather than only moving its markdown file.

## Constraints

- Keep legacy file-form task moves working; this task must not silently promote file tasks to folder form. Promotion belongs to TB-99.
- Treat folder-task contents as owned by the task directory: `TASK.md`, `attachments/`, and the task-local agent paths defined by TB-94/TB-102 move together.
- Hold `.board.lock` for structured task mutation and use the existing atomic-write / regenerate patterns without introducing a second writer for `BOARD.md`.
- Fail on source/destination collisions instead of merging folder contents or overwriting an existing task.
- Do not make board-root `.agent-state/` or `.agent-logs/` the source of truth for folder tasks.

## Acceptance Criteria

- [x] `tb mv`, `tb start`, and `tb done` move a folder-form task by renaming the whole `<status>/<ID>/` directory, preserving `TASK.md`, `attachments/`, and task-local agent artifacts.
- [x] `tb close <ID>` archives a folder-form task as `archive/<ID>/TASK.md` with all task-local contents preserved; restoring from archive via `tb mv <ID> <status>` moves the same directory back.
- [x] Existing file-form tasks still move/archive/restore with the legacy `<status>/<ID>.md` layout and are not promoted by these commands.
- [x] Success and forced-fault CLI tests prove no orphaned task markdown, attachment files, task-local agent artifacts, or duplicate source/destination task entries remain after success or failure.
- [x] Collision tests cover destination file/folder already existing for the same ID and return a clear non-zero error without data loss.
- [x] The filesystem operation exposes one logical task move (no visible copy-then-delete sequence in status directories); TB-105 remains responsible for GUI watcher event coalescing.
- [x] `BOARD.md` is regenerated after a successful move/archive/restore and reflects the new directory state.
- [x] `cd cli && go test ./...` passes with folder-form, file-form, archive, restore, and mixed-board move fixtures.
