# TB-87: Parent epic filter hides the epic card itself

**Type:** bug
**Priority:** P1
**Size:** S
**Module:** gui
**Tags:** gui,frontend,filter,epic-filter,quick-win
**Branch:** —

## Goal

When a user selects an epic in the parent-epic filter, the board should show the selected epic card alongside its child tasks instead of hiding the parent context.

## Context

`FilterBar.svelte` populates the parent-epic dropdown from tasks tagged `epic`, but `gui/frontend/src/lib/filtering.ts` currently applies the selected epic by requiring `task.parent === selectedEpic`. The epic task itself has no parent, so selecting an epic shows its children while the epic card disappears from its own status column.

## Acceptance Criteria

- [x] `applyFilter` treats a selected parent epic as matching both child tasks with `parent === selectedEpic` and the epic task whose `id === selectedEpic`
- [x] The selected epic remains in its real status bucket (`backlog`, `in-progress`, `done`, or `archive` when archive is shown); child tasks remain in their own status buckets
- [x] The parent-epic match composes with the existing type, priority, module, tag, agent, and search filters; only the parent-epic predicate changes
- [x] Add focused Vitest coverage for `gui/frontend/src/lib/filtering.ts` covering an epic plus children and an unrelated task
- [x] Verify with `cd gui/frontend && npm test` and `cd gui/frontend && npm run check`
