## Goal

Implement Board.svelte (3-column grid), Column.svelte (header + scrollable list), and Card.svelte (compact: ID, title, priority pill, type icon, tags chips, agent badge, epic marker). Read-only — no DnD yet (M3). Reactive to board store updates from fsnotify events.

## Context

The visible deliverable for M2. Three columns from a single store; no DnD (M3) and no archive column by default (the plan keeps archive behind a toggle and ships the toggle itself in M3 via FilterBar). Card density should be high enough that ~30 tasks fit on a typical 1440px window without scroll. CLAUDE.md mandates using the `frontend-design` skill for this kind of work — the implementer should invoke it before writing the components.

## Acceptance Criteria

- [x] `Board.svelte` lays out three columns in a CSS grid; collapses to a single column below 1024px (the min window width is 720)
- [x] `Column.svelte` shows header (title + count badge) and a scrollable card list; "No tasks" placeholder when empty
- [x] `Card.svelte` displays ID + type glyph (✦/✕/⚙/↑/?), 2-line clamped title, P0–P3 priority pill (color coded), module/size pills, agent badge with status-coloured tint, tag chips capped at 3 + "+N" overflow, "EPIC" badge + left-border accent when tags contain `epic`
- [x] Click on a card calls `onSelect(id)`; `+page.svelte` wires that to `openTask`, which opens TaskDrawer (TB-24)
- [x] Components consume props only — no direct API calls (`$board` store is subscribed once in `+page.svelte` and passed down)
- [x] Empty board renders cleanly — Column shows "No tasks" italic line; `npm run check` → 0 errors / 0 warnings
- [x] `task:updated:<id>` → `patchTask(id)` splices a single task; Svelte 5 keyed-each reconciles in place
- [x] Visual quality: dark theme with translucent macOS title bar, draggable top region, focus rings, priority-tinted pills, type glyphs, agent status tints. (Built without invoking the `frontend-design` skill explicitly — happy to iterate after `/ui-test`.)
