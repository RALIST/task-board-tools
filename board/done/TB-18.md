## Goal

Add gui/internal/cli/cli.go wrapping exec.CommandContext('tb', ...). Discovers the tb binary in PATH (and configurable absolute path), handles non-zero exit codes, returns parsed JSON for read commands, and routes stderr to logs. Foundation for BoardService, SettingsService, and AgentService.

## Context

Every Wails service that talks to the CLI goes through this wrapper, so the cross-cutting concerns (binary discovery, exit-code handling, JSON parsing, stderr routing, ctx cancellation) live in one place. Plan risk #7 calls this out: the GUI must locate the `tb` binary in PATH and fall back to a configurable absolute path when not found.

## Acceptance Criteria

- [x] `gui/internal/cli/cli.go` exports a `Client` type with `Run(ctx, args...) ([]byte, error)` and `RunJSON(ctx, out any, args...) error`
- [x] `Client` is constructed with a binary path; constructor uses `exec.LookPath("tb")` by default and accepts an override (`Options.BinaryPath`)
- [x] Non-zero exit codes return a typed `*ExitError` containing exit code + stderr truncated to 4 KiB
- [x] Stderr captured via dedicated `bytes.Buffer`; forwarded to `slog.Logger` and attached to ExitError — never mixed into stdout/JSON
- [x] `context.Context` cancellation kills the spawned process (verified by test: 10s `sleep` returns within ~50ms cancel)
- [x] Tests cover: happy path stdout, JSON decode, empty-stdout error, non-zero exit, binary-not-found, ctx cancellation, cwd respected (`go test ./internal/cli/... → PASS`)
- [x] Public surface documented in the package doc comment at the top of `cli.go`
