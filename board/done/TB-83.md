# TB-83: System tray: idle/running glyph + click to show/hide window

**Type:** feature
**Priority:** P2
**Size:** M
**Module:** gui
**Tags:** milestone-m7,gui,tray
**Branch:** —
**Parent:** TB-7

## Goal

Register a system-tray (menu bar on macOS) icon for tb-gui that shows two states — `idle` (no active runs) and `running` (≥1 active run) — and a left-click shows/hides the main window. Lets the user keep agents running in the background without keeping the kanban window in front.

## Context

Per `docs/FEATURES.md` F7.3. State source: subscribe to `agent:run-queued` / `agent:run-started` / `agent:run-finished` / `agent:run-cancelled` Wails events. Maintain a counter of in-flight runs in the tray controller; toggle the icon when the counter crosses zero.

**Probe-first risk**: Wails3 alpha tray API surface is unstable (R1). Start with a static-icon tray that toggles window visibility on click, confirm it appears in the macOS menu bar, *then* wire the dynamic glyph swap and the right-click menu (Show, Settings, Quit).

Two icon assets needed: `tray-idle@2x.png`, `tray-running@2x.png` (template image style, 22pt on macOS). Place under `gui/build/appicon/` alongside the existing app icon. A small `running` dot overlaid on the idle glyph is sufficient — no animation required for M7.

The `ApplicationShouldTerminateAfterLastWindowClosed` setting in `main.go` is currently `true` (last window close quits the app). With a tray, the right behaviour on macOS is to keep the app running. Flip this to `false` only when a tray icon is registered, so the existing no-tray behaviour is unaffected on platforms that don't get a tray.

## Acceptance Criteria

- [x] Tray icon visible in macOS menu bar after app launch
- [x] Left-click toggles main window show/hide
- [x] Right-click (or click on macOS) opens a small tray menu: Show window, Settings…, Quit
- [x] Glyph swaps to `running` style when active-run counter > 0; back to `idle` at 0
- [x] `ApplicationShouldTerminateAfterLastWindowClosed` set to `false` when tray is active so closing the window doesn't quit the daemon
- [x] Counter increments/decrements on `agent:run-started` / `agent:run-finished|cancelled` events
- [x] Probe step documented: static tray icon confirmed working before glyph-swap wiring
- [x] If Wails3 alpha doesn't expose a tray API or it's broken on macOS at the time of implementation: document the gap and defer this child to a post-M7 follow-up; flip the parent's AC to ⊘ for tray
