## Goal

Single source of truth on the frontend for run records. The drawer reads from it; live events patch into it; `AgentRunLog` reads status from it.

## Acceptance Criteria

- [ ] `gui/frontend/src/lib/stores/runs.ts` exports a Svelte store backed by `Map<RunID, Run>`. Every `Run` carries `TaskID` (from TB-49) so the store can index by task without an extra lookup
- [ ] Selector helpers: `runsByTask(taskID)` returns the runs for a task, sorted by `StartedAt` desc (with `QueuedAt` used as the tiebreaker for queued-but-not-started runs)
- [ ] Hydration: when the drawer opens for a task, the orchestrator calls `AgentService.ListRuns(taskID)` and bulk-inserts into the store
- [ ] Wails event handlers (registered once at app start):
  - `agent:run-queued` → insert `Run{RunID, TaskID, Agent, Mode, Status: "queued", QueuedAt: now}`
  - `agent:run-started` → upsert `Run` with `Status: "running"`, `Pid`, `StartedAt: now`
  - `agent:run-log` → no-op for the store (log lines flow through `AgentRunLog`'s own subscription)
  - `agent:run-finished` → patch the matching `Run` with `Status`, `ExitCode`, `FinishedAt: now`
- [ ] **Drawer "Past runs" list rows** (F4.5): each row renders `{relativeTime(StartedAt || QueuedAt)} · {status pill} · {agent}`. Most recent first. Clicking a row sets a local `selectedRunID`, which `AgentRunLog` (TB-51) consumes. Default selection on open: the most recent run, or none if the task has never been run
- [ ] Tests (Vitest): hydrate with 3 runs → selectors return them sorted; emit a `run-queued` event → store grows with queued status; emit `run-started` for the same `run_id` → status patches to running; emit `run-finished` → status patches to terminal; selecting a row updates `selectedRunID`; row rendering shows timestamp + status + agent name
