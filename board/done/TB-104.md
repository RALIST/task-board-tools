## Goal

Drag-and-drop onto a task card and the task drawer must add attachments through tb (never direct file writes), show success/failure feedback, and refresh the task from the board after mutation.

## Context

- @gui/frontend/src/lib/components/Card.svelte — task-card surface; file drops here must coexist with existing card click/keyboard behavior and column card DnD.
- @gui/frontend/src/lib/components/TaskDrawer.svelte — open-drawer surface; reuse the attachment UI/state introduced by TB-103 instead of inventing a separate attachment model.
- @gui/frontend/src/lib/api.ts — frontend import surface for typed Wails calls; add/reuse an attachment wrapper here rather than importing generated bindings directly from components.
- @gui/app/board_service.go and @gui/internal/cli/mutations.go — backend service/CLI wrapper layer that should invoke `tb attach`; the GUI must not copy files into task folders itself.
- @gui/frontend/src/lib/stores/board.ts and @gui/frontend/src/lib/stores/toast.ts — existing refresh and toast patterns to reuse for success/failure feedback.

## Constraints

- Depend on the CLI contract from TB-99: attachment mutation is `tb attach <ID> <path>...`, including legacy file-task auto-promotion and failure atomicity.
- Depend on the drawer attachment model from TB-103: drag-and-drop should feed the same refresh/listing behavior as file-picker add, not create a second attachment path.
- Do not solve watcher coalescing here; TB-105 owns the one-logical-refresh watcher contract. This task should avoid adding duplicate explicit refreshes on top of watcher-driven refresh.
- Preserve existing task-card DnD between columns. File drops should be recognized from `DataTransfer.files`; ordinary card drags must keep moving tasks as before.
- Treat invalid/empty file drops as user-visible failures: show an error toast and leave the current board/drawer state unchanged.

## Acceptance Criteria

- [x] Dropping files on a task card or on the open task drawer calls `tb attach` (never direct file writes).
- [x] Success and failure both surface a toast; failure preserves prior state.
- [x] After mutation the task refreshes from the board with no duplicate refresh events.
- [x] Existing card drag-and-drop between columns still works after file-drop handlers are added.

## Done when

## Manual test (UI)

Drop one file onto a task card; drop multiple files onto an open task drawer; drop an invalid path (e.g. a non-existent file from a synthetic event) and confirm the failure toast. Re-open the drawer and confirm the attachment list reflects only the successful drops.

Also drag a task card between backlog/in-progress/done after adding the file-drop handlers; the status move should still persist and show the normal move failure toast if `tb mv` fails.
