# TB-21: SettingsService: project root, recent boards, folder picker

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** gui
**Tags:** milestone-m2,wails-service,settings
**Branch:** —
**Parent:** TB-2

## Goal

Implement gui/app/settings_service.go with GetProjectRoot, GetBoardInfo, OpenBoard(projectRoot), PickBoardDialog (native Wails3 dialog), and ListRecentBoards (read/write ~/.config/tb-gui/recent.json). On first launch, prompt the folder picker and remember. OpenBoard reloads config, redirects watcher, and emits board:reloaded.

## Context

GUI needs to know which board it is showing. The plan settles this clearly (risk #4): `OpenBoard` accepts the project root (the directory containing `.tb.yaml`), not the board directory itself, so the existing `loadProjectConfig` walk works the same way the CLI does. The recent-boards file is small persistent state — keep it in the OS config dir, not inside any board.

## Acceptance Criteria

- [x] `gui/app/settings_service.go` declares `SettingsService` with `GetProjectRoot`, `GetBoardInfo`, `OpenBoard(ctx, projectRoot)`, `PickBoardDialog`, `ListRecentBoards`
- [x] `BoardInfo` carries `{ProjectRoot, BoardDir, Prefix, WIPLimit}` parsed from `.tb.yaml` (mirrors `cli/board.go`)
- [x] `OpenBoard` validates `.tb.yaml`, swaps the CLI client into BoardService, calls watcher.Switch, prepends to recent, emits `board:opened` + `board:reloaded`. On watcher failure no state is committed (tested).
- [x] `PickBoardDialog` uses `application.Get().Dialog.OpenFile().CanChooseDirectories(true).PromptForSingleSelection()`; returns `ErrCancelled` on dismiss
- [x] `RecentBoard` shape `{projectRoot, prefix, lastOpened}`; stored as JSON at `$XDG_CONFIG_HOME/tb-gui/recent.json` (fallback `~/.config/tb-gui/recent.json`); atomic write via temp+rename
- [x] Dedup by `projectRoot`; cap at 20; sorted by `lastOpened` desc
- [ ] On app start, if no recent or last one has no `.tb.yaml`, the dialog opens automatically — frontend behaviour, covered by TB-22 (the SettingsService surface exposes everything the frontend needs)
- [x] `ErrNoTbYaml` returned for paths without `.tb.yaml`; frontend treats it as a toast (tested)
- [x] Both `BoardService` + `SettingsService` registered in `main.go` and visible in generated bindings (`wails3 generate bindings` → 2 Services, 7 Methods, 6 Models)
