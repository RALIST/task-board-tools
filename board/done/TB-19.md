# TB-19: BoardService: LoadBoard and GetTask via exec tb

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** gui
**Tags:** milestone-m2,wails-service
**Branch:** —
**Parent:** TB-2

## Goal

Implement gui/app/board_service.go with LoadBoard() (exec tb ls --json --status active) and GetTask(id) (exec tb show <id> --json). Exposed as Wails bindings. Returns typed structs that the frontend can consume directly. No mutations in M2 — those land in M3.

## Context

First Wails service exposed to the frontend. Read-only by design — every mutation in M2 stays in the CLI's lane. The board snapshot returned must match the JSON shape that TB-12 defines so the frontend can use it directly without an adapter layer.

## Acceptance Criteria

- [x] `gui/app/board_service.go` declares `BoardService` with `LoadBoard(ctx) (BoardSnapshot, error)` and `GetTask(ctx, id string) (TaskDetail, error)`
- [x] `LoadBoard` shells out via `cli.Client.RunJSON(... "ls", "--json", "--status", "active")` and returns the parsed snapshot, bucketed server-side into `Backlog` / `InProgress` / `Done` (archive dropped — surfaces in M3 behind a filter)
- [x] `GetTask` shells out via `... "show", id, "--json"` and returns `{metadata, body}`
- [x] Service registered with Wails via `application.NewService(boardService)` in `main.go`; generated bindings expose `LoadBoard` + `GetTask` (`setClient` kept unexported so it's not in the TS surface)
- [x] Types mirror the CLI's JSON contract (TB-12) with `json:""` tags; bindings emit `Task`, `BoardSnapshot`, `TaskDetail`
- [x] Empty board → non-nil empty slices in every column (verified by `TestLoadBoard_EmptyArray`)
- [x] Unknown ID → `ErrNotFound` (mapped from CLI stderr `"not found in any directory"`); verified by `TestGetTask_NotFound`
