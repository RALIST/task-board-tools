# TB-49: AgentService.ListRuns: parse JSONL into Run summaries

**Type:** feature
**Priority:** P1
**Size:** S
**Module:** gui
**Tags:** milestone-m4,agent,wails-service,history
**Branch:** —
**Parent:** TB-4

## Goal

Roll up the per-task JSONL into one `Run` record per `run_id` so the drawer can show a past-run list.

## Acceptance Criteria

- [ ] `AgentService.ListRuns(ctx, id) ([]Run, error)` reads `board/.agent-state/<ID>.jsonl` line-by-line, groups events by `run_id`, and emits one `Run{RunID, TaskID, Agent, Mode, QueuedAt, StartedAt, FinishedAt, Status, ExitCode, LogPath}` per run. `TaskID` is required so `GetRunLog` can reconstruct the log file path without an external index
- [ ] Sorted by `StartedAt` desc (most recent first); a queued-but-never-started run sorts by `QueuedAt`
- [ ] Tolerates a trailing partial JSON line (writer is append-only and may not have flushed); skip-with-warning rather than fail the whole call
- [ ] `LogPath` is the absolute path `board/.agent-logs/<ID>/<run_id>.log` regardless of whether the file exists yet (drawer handles the missing-file render)
- [ ] Missing JSONL file returns `[]Run{}` (not nil, not an error) — first-time view of a task should show an empty list cleanly
- [ ] Wails bindings regenerated; the `Run` type is visible from the frontend
- [ ] `AgentService.GetRunLog(taskID, runID string) (string, error)` reads `board/.agent-logs/<taskID>/<runID>.log` and returns the contents; returns a typed `ErrNotFound` when the file is missing (used by TB-51 to render past runs). Takes both IDs explicitly so the frontend doesn't need to resolve `runID → taskID`
- [ ] Tests: fixture with 3 well-formed runs in mixed order → sorted by `StartedAt` desc; fixture with a trailing half-line → 3 runs returned, no error; missing file → empty slice; `GetRunLog` round-trip; `GetRunLog` on missing file → `ErrNotFound`
