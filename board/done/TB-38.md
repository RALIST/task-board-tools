# TB-38: FilterBar with archive column toggle

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** gui
**Tags:** milestone-m3,frontend,filter
**Branch:** —
**Parent:** TB-3

## Goal

Client-side filtering over the loaded snapshot: type, priority, module, tags (multi), parent epic, agent. 'Show archived' toggle adds an Archive column.

## Acceptance Criteria

- [ ] `FilterBar.svelte` component above the kanban with controls: type (multi), priority (multi), module (multi from observed values), tags (multi), parent epic (single), agent (multi)
- [ ] Filters compose as AND across categories, OR within a category
- [ ] Filters live in `$lib/stores/filter.ts` (extend the existing store; current store already exists)
- [ ] Filtering is purely client-side over the loaded board snapshot — no extra `tb` calls
- [ ] "Show archived" toggle: when on, the frontend calls `BoardService.LoadBoard` in `all` mode (TB-40) and renders an Archive column to the right of Done; when off, returns to `active` mode and the Archive column disappears
- [ ] Depends on TB-40 for the backend `LoadBoard` archive-aware mode and the `archive` bucket in `BoardSnapshot`
- [ ] "Clear filters" button resets to default state
- [ ] Filter state persists across `board:reloaded` events
- [ ] Empty filter result renders a clear empty-state message per column
