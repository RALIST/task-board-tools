# TB-32: BoardService: CreateTask, EditTask, MoveTask, CloseTask, Regenerate via exec tb

**Type:** feature
**Priority:** P1
**Size:** M
**Module:** gui
**Tags:** milestone-m3,wails-service,mutations
**Branch:** —
**Parent:** TB-3

## Goal

Add Wails-bound service methods that exec the CLI wrappers for all structured mutations (F3.1 backend, F3.2, F3.3, F3.6). Errors return typed values consumable by the frontend toast store.

## Acceptance Criteria

- [ ] `BoardService.CreateTask(input)` → CLI `tb create`; returns the created task ID by way of the TB-31 wrapper (`Created <path>` → extracted ID)
- [ ] `BoardService.EditTask(id, fields)` → CLI `tb edit`; supports priority, type, size, module, tags
- [ ] `BoardService.MoveTask(id, status)` → CLI `tb mv`; status accepts `backlog | in-progress | done`
- [ ] `BoardService.CloseTask(id)` → CLI `tb close`
- [ ] `BoardService.Regenerate()` → CLI `tb regenerate`
- [ ] All methods bind cleanly through Wails and are callable from the frontend `api.ts`
- [ ] Errors surface as typed Go errors mappable to `errorString()` in the frontend
- [ ] Unit tests against a fake `tb` cover happy path + one error per method
- [ ] Depends on TB-31
