# TB-61: Stale-recovery cancelled carve-out: never overwrite AgentStatus=cancelled

**Type:** feature
**Priority:** P1
**Size:** S
**Module:** gui
**Tags:** milestone-m5,agent,daemon,cancel-stick
**Branch:** —
**Parent:** TB-5

## Goal

Recovery (TB-60) only looks at `AgentStatus=running` tasks, so `cancelled` should never enter the scan in the first place — but a partial cancel where the JSONL `cancelled` event was written and the AgentStatus write was interrupted is still possible. Add a defensive carve-out: if at any point during recovery the task's *latest JSONL event* is `finished{status: cancelled}`, treat the task as cancelled, set `AgentStatus=cancelled`, and never set it to `failed`. Document this as the dual of the M4 cancel ordering invariant.

## Context

The M4 cancel ordering (TB-48, 5 steps): mark → kill → JSONL `finished{cancelled}` → Wails emit → `tb edit --agent-status cancelled`. If the GUI is `kill -9`ed between steps 3 and 5, the JSONL says `cancelled` but the task `.md` still says `running`. Recovery would naively treat this as a stale-running and synthesize `finished{failed}` — overwriting the user's cancellation. That is the bug R12 calls out.

Rule: **JSONL is the durable record of intent.** If the JSONL last event is `cancelled`, the user wanted cancel, and we honor it during recovery by reconciling AgentStatus to cancelled (not failed).

Code shape: during TB-60's per-task loop, after parsing JSONL, inspect the latest event. If it is `finished{status: cancelled}`, take the cancelled branch (write `tb edit --agent-status cancelled`, emit `agent:run-finished{cancelled}`) and skip the failed-synthesis path.

Also document the invariant in `docs/ARCHITECTURE.md` (in "Cancel carve-out" section, line ~202).

- [ ] Recovery loop (TB-60) inspects the latest JSONL event per stale task before deciding outcome
- [ ] If latest event is `finished{status: cancelled}` → reconcile `AgentStatus=cancelled` via `tb edit`; do NOT append a new synthetic event
- [ ] If task's *current* `AgentStatus` is already `cancelled`, recovery is a no-op (defensive — should never enter the loop, but assert)
