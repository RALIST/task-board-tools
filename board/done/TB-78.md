# TB-78: Wire cli_path preference into cli.NewClient at board open + reload on change

**Type:** feature
**Priority:** P2
**Size:** S
**Module:** gui
**Tags:** milestone-m7,settings,backend
**Branch:** —
**Parent:** TB-7

## Goal

Persist a user-specified `tb` binary path in `preferences.json` (TB-76) and feed it to `cli.NewClient` at board-open time, plus rebuild the active client when the user updates `cli_path` from the settings panel — so the GUI works on systems where `tb` isn't on PATH.

## Context

`cli.NewClient` already accepts a `BinaryPath` override (`gui/internal/cli/cli.go:39`), and `SettingsOptions.CLIPath` plumbs it through `SettingsService` for tests. The construction-time `s.cliPath` is empty in production today, so the GUI relies on `tb` being on PATH (R8). What's missing is the persistence + reload loop:
1. `SettingsService.OpenBoard` (`gui/app/settings_service.go:159–161`) currently passes `s.cliPath` to `cli.NewClient`. Switch that source to `s.GetCLIPath()` (with empty falling back to PATH lookup, matching today's behaviour).
2. `SetCLIPath(string)` is the live-reload hook. After persisting the new value, it snapshots the active `BoardInfo`, builds a fresh `cli.Client`, and calls `s.board.setClient(newClient)` (`gui/app/board_service.go:90`) — no reopen, no watcher restart. Guard against a SetCLIPath while no board is open: skip the `setClient` step in that case.
3. On `ErrBinaryNotFound`, surface a toast via the existing error path; do NOT roll back the preference (let the user fix it interactively in the panel).

## Acceptance Criteria

- [x] `SettingsService.OpenBoard` reads `GetCLIPath()` and passes it as `cli.Options.BinaryPath`
- [x] `SetCLIPath(string) error` (added in TB-76) is extended here: persists the new value, snapshots the active `BoardInfo`, builds `cli.NewClient` with the new path, then calls `b.setClient(newClient)` if a board is currently open. No-op for `setClient` when no board is open.
- [x] Test: with `cli_path` pointing at a stub `tb`, `LoadBoard` invokes the stub; flip `cli_path` to a different stub via `SetCLIPath`, next `LoadBoard` invokes the new stub — no reopen
- [x] Empty `cli_path` falls back to `exec.LookPath("tb")` (current behaviour preserved)
- [x] `cli.ErrBinaryNotFound` from a bad `cli_path` propagates as a typed error so the frontend can show a friendly toast (R8 in `IMPLEMENTATION.md`)
