# TB-26: Atomic write for .next-id under board lock

**Type:** tech-debt
**Priority:** P2
**Size:** S
**Module:** cli
**Tags:** durability

## Goal

tb create allocates by writing .next-id with plain os.WriteFile under flock. A truncated write leaves the counter unparsable and blocks future creates. Use writeFileAtomic for .next-id too.
