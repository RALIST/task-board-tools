## Goal

Implement the CLI-side removal path for task attachments: `tb attach --rm <ID> <attachment-name>...` removes existing attachments from the task's canonical `attachments/` directory, updates the task markdown, and refuses any name/path that could delete outside that directory.

## Context

- Parent epic **TB-93** defines the file-form / folder-form coexistence model and requires folder tasks to own user attachments under the task directory.
- **TB-94** owns the architecture contract for canonical folder-task paths (`TASK.md`, `attachments/`, lock/atomicity rules). If that contract changes, follow it here.
- **TB-99** owns the add/promote side of `tb attach`; reuse the same command parsing shape, attachment list format, locking helpers, and Log wording where possible.
- **TB-103** will call this command from the GUI task drawer for attachment removal, so stderr and exit codes must be stable enough for a wrapper to surface user-facing failures.

## Constraints

- Scope is CLI/storage only; do not add GUI behavior in this task.
- Run validation, task markdown update, and deletion under `.board.lock`.
- Treat removal arguments as attachment names under the canonical attachment directory, not arbitrary filesystem paths. Reject empty names, absolute paths, `..`, path separators, and anything that resolves outside `<status>/<ID>/attachments/`.
- Prefer all-or-nothing behavior for multi-name removal: validate every requested attachment before deleting any file or changing task markdown.
- Do not auto-promote legacy file-form tasks during removal. If a task has no folder-form attachment directory, fail clearly and leave the board unchanged.

## Acceptance Criteria

- [x] `tb attach --rm <ID> <name>...` locates a folder-form task in any board status and removes the listed attachments under `.board.lock`.
- [x] Before deletion, the command validates all requested names; missing attachments, empty names, absolute paths, `..`, path separators, symlink/resolution escapes, or non-folder-form/no-attachment-dir targets return non-zero, write a clear reason to stderr, and leave attachments plus task markdown unchanged.
- [x] Successful removal deletes the attachments, removes their entries from the task's `## Attachments` section, appends one task Log entry listing the removed names, and refreshes generated board output through the normal CLI mutation path.
- [x] `cli/` tests cover single and multi-attachment success, missing attachment, legacy file-form/no-attachment-dir target, traversal/path-separator attempt, outside-resolution attempt, and lock contention.
