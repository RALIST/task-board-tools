# tb-gui

Wails3 + SvelteKit desktop kanban for the `tb` CLI.

This module is part of `task-board-tools`. See [`../docs/PROJECT.md`](../docs/PROJECT.md) for product context and [`../docs/ARCHITECTURE.md`](../docs/ARCHITECTURE.md) for the GUI design.

## Status

M2 scaffold — read-only kanban is in progress under epic `TB-2`. See [`../docs/IMPLEMENTATION.md`](../docs/IMPLEMENTATION.md).

## Prerequisites

- Go `1.26.2+` (`go version`)
- Wails CLI `v3.0.0-alpha.91` (`wails3 version`)
- Node 20+ with `npm` (or `pnpm`)
- CGo toolchain: `gcc` or `clang`; on macOS install Xcode CLI tools (`xcode-select --install`)

Confirm with `wails3 doctor`.

## Develop

```
# from gui/ — generates ./frontend/bindings/ (gitignored)
wails3 generate bindings -clean -ts

# install frontend deps once
(cd frontend && npm install)

# dev mode (hot reload, opens a window)
wails3 dev
```

## Build

```
# production build → ./bin/tb-gui (and a .app on macOS via wails3 build)
wails3 build
```

To produce a debug binary directly with Go (no `.app` wrapper):

```
(cd frontend && npm run build) && go build ./
```

The frontend `dist/` must exist before `go build` because `main.go` embeds it.

## Layout

| Path | Purpose |
|------|---------|
| `main.go` | App entry; window options; single-instance plugin |
| `app/` | Wails services exposed to the frontend (M2: empty; M2 sub-tasks add `BoardService`, `SettingsService`) |
| `internal/` | Non-exported Go helpers (M2 adds `cli/`, `watcher/`; M4 adds `agent/`; M5 adds `daemon/`) |
| `frontend/src/` | SvelteKit app (Svelte 5 + TS) |
| `frontend/bindings/` | Auto-generated Wails TS bindings (do not edit; regenerated by `wails3 generate bindings`) |
| `build/` | Wails build config + platform-specific assets |
